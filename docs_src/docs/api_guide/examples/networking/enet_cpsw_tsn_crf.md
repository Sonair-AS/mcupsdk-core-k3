# Ethernet CRF Master Example {#EXAMPLES_ENET_CPSW_TSN_CRF_MASTER}

[TOC]

# Introduction

\cond SOC_AM62DX
\note This example supports the AM62D-EVM-PROC180E2. To test with the AM62D-EVM-PROC180E1, change the phy addresses in TI BOARD DRIVERS -> ETHPHY -> CONFIG_ENET_ETHPHY(x) -> phy Address to 15 and 3.
\endcond

This example demonstrates the CRF Talker using IEEE Std 1722™-2016 stack with CPSW Peripheral.

In this example, the DUT MAC port is connected to a PC which captures the CRF packets from the talker to verify the functionality of the CRF Talker.

See also :\ref ENET_CPSW_AVTP

# Prerequisites
- @VAR_BOARD_NAME_LOWER board with dp83tg721 add-on card.
- MDI Media Converter (Required only for this demo, not required for actual CRF Usecase)
- PC with Linux Ubuntu OS with wireshark and linux-ptp installed.

# Steps to Run the Example

## Build the example

Refer \ref EXAMPLES_ENET_CPSW_TSN_GPTP to build the avtp examples.

## HW Setup

- Connect the @VAR_BOARD_NAME_LOWER-dp83tg721 board's Macport 1 to the media convertor and the media convertor to PC's ethernet port using CAT6 cable.

## Run the example

\attention If you need to reload and run again, a CPU power-cycle is MUST

- Start ptp4l on PC as master. Refer to \ref EXAMPLES_ENET_CPSW_TSN_GPTP to setup PC as gptp master.
- Open wireshark on PC to capture the CRF packets.
- Load the CRF Talker example to the DUT and click on Run.
- Save the captured packets on wireshark.
- Look for the "INF:avtp:CRF talker 00:01:02:03:04:05:00:04:pdus/sec=49 freq_Hz=48000 ts_interval=160" print in the logs, this confirms the generation of CRF Frames.
- The Media clock generated by the DP83TG721 phy, can be measured on the TP10 testpoint of the add-on card.
